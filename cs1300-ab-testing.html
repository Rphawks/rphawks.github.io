OVERVIEW

the a/b testing done here seeks to work with a very basic data set and extrapolate whether interface changes, including logical arrangement and contrast improvement, reduce user interaction workload and error. though the scale for this dataset is incredibly small, in theory the statistical analyses applied here would apply to any scale, allowing for a designer to make more concrete conclusions about the effects of interface changes.

WEBSITE CHANGES

the appointment scheduling interface above was changed to group appointments visually by location and ordered by first appointment available for each location. the scheduling buttons were arranged to allow for distinction by type and the backgrounds darkened for contrast.

METRIC & HYPOTHESES

the additional chosen metric is mouse movement distance.

misclick rate:
null: users will have no changes in misclick rate, regardless of the contrast of interface elements.

alternative: users will have a lower misclick rate if interface elements are high contrast.

prediction: the data will reject the null hypothesis, as users will have a lower misclick rate with higher contrast. this is because the higher contrast will make the text more readable, leading to lower misclicking.


time on page:
null: time on page is unaffected by the organization of the user interface elements.

alternative: time on page is lower if the user interface elements are organized logically.

prediction: the data will reject the null hypothesis, as a logical organization of elements should reduce cognitive workload and allow for faster decision making. such organization including grouping by location and date using gestalt principles of proximity or continuity should facilitate these improvements.


mouse movement distance:
null: mouse movement distance is unaffected by the organization of the user interface elements.

alternative: mouse movement distance is lower if the user interface elements are organized logically.

prediction: the data will reject the null hypothesis, as, again, a logical organization of elements should allow for faster decision making, as less searching will be needed, leaning to less mouse movement.


STATISTICAL ANALYSIS

misclick rate:
test used: a chi-squared test was used, with group a having higher contrast than group b. the category tested for was misclick (or not).

results: df = 1, chi^2 = 1.35, p-value = 0.24

conclusion: the p-score is too high to indicate any conclusive result. it fails to reject the null hypothesis. this could likely be due to small sample size and familiarity with the interface, as in both groups, the vast majority of people did not misclick.


time on page:
test used: a one-tailed t-test was used, with baseline group a having randomized order, relative group b having logical organization.

results: avg(a) = 12860, avg(b) = 20230, dof = 31.30,
t-score = -0.855, p-value = .19

conclusion: the p-score is too high to indicate any conclusive result, even though the time with organization (group b) was higher. it fails to reject the null hypothesis.
this could be due to the fact that the button was further down the page in the organized version.

mouse movement distance:
test used: a one-tailed t-test was used, with baseline group a having randomized order, relative group b having logical organization.

results: avg(a) = 4006, avg(b) = 2921, dof = 43.57, t-score = -2.31,
p-value = 0.01

conclusion: with an acceptably low p-score <=0.05, there is significant difference between the two groups. because the average of b is lower, we know that the group with logical organization has a statistically significant lower mouse movement, which rejects the null hypothesis and gives us confident the alternative hypothesis is true. more organization then does correlate with less mouse movement.


STATISTICS DISCUSSION
group a:
on misclicks, group a performed worse anecdotally with more misclicks at seven misclicks, but not enough to be statistically significant, with a chi-squared test p-value of 0.24. surprisingly, group a, which was not organized, had a lower average time on page at 12.8s compared to 20.2s, though it was not statistically significant either. this could be due to the nature of the task and that, by coincidence, the button required to complete the task was higher on the random order version a. what was statistically significant was that group a had substantially higher mouse movement distance (4006) than group b (2921).

group b:
group b had qualtiatively fewer misclicks but a longer time on page, but again both were not significant. group b did have much lower mouse movement. given our one statistically significant metric, more testing would perhaps be required to make concrete decisions, but the one test we do have—mouse movement distance, which could be argued is related to cognitive workload— is significantly in group b's favor. version b used gestalt organization to provide a better conceptual model by location and time, and used higher contrast for accessibility. by design principles and this very light statistical testing, it is the preferred page.
