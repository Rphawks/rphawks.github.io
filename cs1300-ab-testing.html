A/B TESTING
WORK TYPE  solo
CONTEXT    csci1300, brown university
TIME       8 hours
OVERVIEW

the a/b testing done here seeks to work with a very basic data set and extrapolate whether interface changes, including logical arrangement and contrast improvement, reduce user interaction workload and error. though the scale for this dataset is incredibly small, in theory the statistical analyses applied here would apply to any scale, allowing for a designer to make more concrete conclusions about the effects of interface changes.

WEBSITE CHANGES

the appointment scheduling interface above was changed to group appointments visually by location and ordered by first appointment available for each location. the scheduling buttons were arranged to allow for distinction by type and the backgrounds darkened for contrast.

METRIC & HYPOTHESES

the additional chosen metric is mouse movement distance.

misclick rate:
null: users will have a higher misclick rate if interface elements are high contrast.

alternative: users will have a lower misclick rate if interface elements are high contrast.

prediction: users should have a lower misclick rate if interface elements are high contrast, though it may not be significant.


time on page:
null: time on page is lower if the user interface elements are organized logically.

alternative: time on page is unaffected if the user interface elements are organized logically.

prediction: time on page should be lower if the user interface elements are organized logically.


mouse movement distance:
null: misclicking leads to significantly increased mouse movement distance.

alternative: misclicking does not significantly affect mouse movement distance.

prediction: misclicking may not significantly affect mouse movement distance at all.


STATISTICAL ANALYSIS

misclick rate:
test used: a two-tailed t-test was used, with group a having higher contrast than group b. this is because the two hypotheses are on both sides of a standard (higher and lower).

results: avg(a) = .20, var(a) = .16, avg(b) = .10, var(b) = .09, dof = 60.17, t-score = -1.12, p-value = .26

conclusion: the p-score is too high to indicate any conclusive result, even though the average misclick rate of group b was lower. it fails to reject the null hypothesis.


time on page:
test used: a one-tailed t-test was used, with baseline group a having randomized order, relative group b having logical organization.

results: avg(a) = 12860, avg(b) = 20230, dof = 31.30, t-score = -0.855, p-value = .19

conclusion: the p-score is too high to indicate any conclusive result, even though the time with organization (group b) was higher. it fails to reject the null hypothesis.
this could be due to the fact that the button was further down the page in the organized version.

mouse movement distance:
test used: a one-tailed t-test was used, with baseline group a being non-misclicking, relative group b misclicking.

results: avg(a) = 5946, avg(b) = 3028, dof = 10.39, t-score = 2.76, p-value = 0.99

conclusion: the p-score is far too high to indicate any conclusive result. it fails to reject the null hypothesis.
